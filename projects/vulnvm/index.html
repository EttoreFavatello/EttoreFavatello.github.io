Vulnerable VM

Laboratorio Offensive Security in stile HackTheBox / VulnHub. Macchina virtuale deliberatamente vulnerabile progettata per essere compromessa attraverso una catena di attacco realistica composta da tre fasi.

[ ← home ]

// Overview

Progetto di sistemi operativi e networking sviluppato durante il percorso ITS in cybersecurity. Ho costruito da zero una macchina virtuale deliberatamente vulnerabile, configurata in modo da riprodurre uno scenario realistico di compromissione: dall'enumerazione iniziale fino al privilege escalation finale tramite un exploit kernel.

// Setup del laboratorio

L'ambiente è basato su virt-manager con backend KVM/QEMU — un hypervisor di tipo 1 integrato nel kernel Linux — su cui ho configurato due macchine virtuali isolate in una rete NAT interna:

  • Kali Linux — postazione attaccante
  • Ubuntu Server 20.04.1 LTS — bersaglio

La VM vittima monta il kernel 5.8.0-63-generic, vulnerabile al CVE-2022-0847 (Dirty Pipe) — una falla del meccanismo delle pipe che consente a un utente non privilegiato di sovrascrivere file di sola lettura e ottenere privilegi root. Per mantenere attive le vulnerabilità ho disabilitato le repository apt focal-security, impedendo l'installazione automatica delle patch.

// Vulnerabilità introdotte

Sul server ho installato uno stack LAMP (Apache, MySQL, PHP) con WordPress 6.2 e il plugin Memphis Document Library 3.1.5, affetto da una vulnerabilità di Arbitrary File Download. Ho inoltre piazzato:

  • Una backdoor PHP nella webroot (esecuzione comandi via parametro URL)
  • Uno script backup.sh contenente credenziali in chiaro, per simulare la negligenza di un sysadmin reale

// Catena di attacco

01
Foothold
L'attaccante sfrutta il plugin WordPress vulnerabile o la backdoor PHP per ottenere esecuzione di codice, atterrando come utente www-data.
02
Lateral Movement
Recupera le credenziali esposte nello script backup.sh per autenticarsi come sysadmin e leggere il flag user.txt.
03
Privilege Escalation
L'utente normale non ha sudo. L'unica via è l'exploit Dirty Pipe (CVE-2022-0847) sul kernel 5.8 per ottenere root e leggere root.txt.

// Tool utilizzati

tool
nmap
Ricognizione iniziale dalla Kali. Ha rivelato le porte aperte (22 SSH, 80 HTTP) e il sistema operativo della VulnMachine.
tool
gobuster
Enumerazione di file e directory nascosti nella webroot. Ha trovato /backdoor.php, /backup.sh e /wordpress.
tool
curl
Download del contenuto di backup.sh dalla webroot, rivelando le credenziali in chiaro di sysadmin.
tool
Burp Suite
Scoperta del parametro corretto della backdoor PHP. Tramite Intruder ha testato 12 nomi comuni identificando "cmd" dall'analisi delle risposte.
tool
revshells.com
Generazione del payload reverse shell Python3 con IP e porta precompilati e URL encoding automatico.
tool
netcat (nc)
In modalità listener (nc -lvnp 4444) sulla Kali per ricevere la connessione della reverse shell dalla VulnMachine.
tool
Python3 pty
Stabilizzazione della reverse shell in una TTY completa con tab completion, Ctrl+C e history.
tool
SSH
Lateral movement con le credenziali trovate in backup.sh (sysadmin/sysadmin123) per autenticarsi sulla VulnMachine.
tool
gcc
Compilatore C usato direttamente sulla VulnMachine per compilare il sorgente dell'exploit Dirty Pipe.
tool
Dirty Pipe exploit
Exploit C (Arinerron, GitHub) per CVE-2022-0847. Sfrutta il meccanismo delle pipe del kernel 5.8 per modificare /etc/passwd e ottenere shell root.
tool
Python3 HTTP server
Server web temporaneo (python3 -m http.server 8000) sulla Kali per trasferire dirtypipe.c alla VulnMachine via HTTP.
